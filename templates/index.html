🤖 API Chatbot Royal's Resto

Dokumentasi lengkap API untuk Chatbot Royal's Resto. Base URL: http://localhost:5000

🔗 Base URL: http://localhost:5000 (development) / https://api.royalsresto.com (production)

💬 Chatbot Endpoints

1. Kirim Pertanyaan ke Chatbot

POST /chat Public

Deskripsi: Mengirim pertanyaan ke chatbot dan mendapatkan jawaban.

Request Body:

{
    "pertanyaan": "Apa menu favorit di Royal's Resto?"
}

Response (Jawaban Normal - status ok):

{
    "pertanyaan": "Apa menu favorit di Royal's Resto?",
    "jawaban": "Menu favorit kami adalah Kepiting Kenari dan Dendeng Rusa.",
    "status": "ok"
}

Response (Pertanyaan Ambigu - status ambigu):

{
    "pertanyaan": "resto",
    "opsi_pertanyaan": [
        "Apa jam buka Royal's Resto?",
        "Dimana lokasi Royal's Resto?",
        "Apa saja menu di Royal's Resto?"
    ],
    "jawaban": "Pertanyaan mana yang kamu maksud?",
    "status": "ambigu"
}

Response (Pertanyaan Tidak Dikenal - status unknown):

{
    "pertanyaan": "pertanyaan acak",
    "jawaban": "Mohon maaf, saya belum mengerti pertanyaan Anda.",
    "status": "unknown"
}

🔐 Authentication Endpoints

1. Login Admin

POST /login Public

Deskripsi: Login untuk mendapatkan token akses admin.

Request Body:

{
    "username": "admin",
    "password": "admin123"
}

Response (Sukses):

{
    "message": "Login berhasil",
    "authenticated": true,
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "admin": {
        "id": 1,
        "username": "admin",
        "email": "admin@royalsresto.com",
        "full_name": "Administrator",
        "role": "super_admin"
    },
    "expires_in": 86400
}

2. Logout Admin

POST /logout Auth Required

Deskripsi: Logout dan menghapus session admin.

Headers:

Authorization: Bearer <token>

Response:

{
    "message": "Logout berhasil",
    "authenticated": false
}

3. Verifikasi Token

GET /verify-token Public

Deskripsi: Memverifikasi apakah token masih valid.

Headers:

Authorization: Bearer <token>

Response:

{
    "authenticated": true,
    "admin_id": 1,
    "username": "admin",
    "role": "super_admin"
}

4. Ganti Password Admin

POST /change-password Auth Required

Deskripsi: Mengganti password admin yang sedang login.

Headers:

Authorization: Bearer <token>

Request Body:

{
    "old_password": "admin123",
    "new_password": "newpassword123",
    "confirm_password": "newpassword123"
}

Response:

{
    "message": "Password berhasil diubah"
}

5. Profil Admin

GET /admin-profile Auth Required

Deskripsi: Mendapatkan profil admin yang sedang login.

Headers:

Authorization: Bearer <token>

Response:

{
    "admin": {
        "id": 1,
        "username": "admin",
        "email": "admin@royalsresto.com",
        "full_name": "Administrator",
        "role": "super_admin",
        "last_login": "2024-01-15 10:30:00",
        "created_at": "2024-01-01 00:00:00"
    }
}

👥 Admin Management Endpoints (Super Admin Only)

1. Daftar Semua Admin

GET /api/admins Super Admin Only

Deskripsi: Mendapatkan daftar semua admin (hanya super admin).

Headers:

Authorization: Bearer <token>

Query Parameters:

  • page (optional) - Nomor halaman, default: 1
  • per_page (optional) - Jumlah per halaman, default: 10
  • search (optional) - Pencarian username/email/nama

Response:

{
    "page": 1,
    "per_page": 10,
    "total_data": 5,
    "total_pages": 1,
    "data": [
        {
            "id": 1,
            "username": "admin",
            "email": "admin@royalsresto.com",
            "full_name": "Administrator",
            "role": "super_admin",
            "is_active": true,
            "last_login": "2024-01-15 10:30:00",
            "created_at": "2024-01-01 00:00:00"
        }
    ]
}

2. Tambah Admin Baru

POST /api/admins Super Admin Only

Request Body:

{
    "username": "admin_baru",
    "password": "password123",
    "email": "admin_baru@royalsresto.com",
    "full_name": "Admin Baru",
    "role": "admin",
    "is_active": true
}

Response:

{
    "message": "Admin berhasil dibuat",
    "admin_id": 2

Request Body:

{

ℹ️ Informasi Tambahan

  • Authentication: Gunakan header Authorization: Bearer <token> untuk endpoint yang membutuhkan auth.
  • Token Expired: Token akan kadaluarsa setelah JWT_EXPIRATION_HOURS (default 24 jam).
  • Role: Super Admin memiliki akses penuh, Admin biasa hanya akses terbatas.
  • Dataset: Data disimpan dalam file CSV di folder data/dataset.csv.
  • Model: Model disimpan dalam file pickle di folder model/model_qa.pkl.
  • Unknown Questions: Pertanyaan yang tidak dikenali otomatis disimpan ke database.
  • Pagination: Gunakan parameter page dan per_page untuk mengatur pagination.
