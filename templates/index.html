🤖 API Chatbot Royal's Resto

Dokumentasi lengkap API untuk Chatbot Royal's Resto.

🔗 Base URL Development: http://localhost:5000
🔗 Base URL Production: https://royals-resto-bot-api.vercel.app
📌 Catatan Penting:
  • Semua endpoint (kecuali /login, /health, /) memerlukan header X-API-Key untuk keamanan.
  • Endpoint yang membutuhkan autentikasi admin juga memerlukan header Authorization: Bearer <token> (diperoleh dari /login).
  • Role super_admin dapat mengelola semua data, sedangkan admin hanya dapat melihat/edit profil sendiri.
  • Karena API berjalan di serverless (Vercel), operasi berat seperti training model sebaiknya dilakukan secara lokal.

🔐 Autentikasi & Otorisasi

API Key (X-API-Key)

Semua endpoint (kecuali /, /health, /login) memerlukan API Key yang dikirim melalui header X-API-Key. Nilai default: RoyalsResto2024SecureKey!@#$. (Dapat diubah di environment variable API_KEY)

JWT Token (Bearer)

Endpoint yang membutuhkan hak akses admin (misalnya /api/admins, /admin-profile, dll.) memerlukan token JWT yang didapat dari endpoint /login. Token dikirim dalam header: Authorization: Bearer <token>.

1. Login Admin

POST /login Public (tanpa API Key)

Request Body:

{
    "username": "admin",
    "password": "admin123"
}

Response Sukses (200):

{
    "authenticated": true,
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "admin": {
        "id": 1,
        "username": "admin",
        "email": "admin@royalsresto.com",
        "full_name": "Administrator",
        "role": "super_admin"
    },
    "expires_in": 86400
}

Response Gagal (401): {"error": "Username atau password salah", "authenticated": false}

2. Logout

POST /logout Auth Required

Headers: Authorization: Bearer <token> + X-API-Key

Response: {"message": "Logout berhasil", "authenticated": false}

3. Verifikasi Token

GET /verify-token Auth Required

Response: {"authenticated": true, "admin_id": 1, "username": "admin", "role": "super_admin"}

4. Ganti Password

POST /change-password Auth Required

Request Body:

{
    "old_password": "admin123",
    "new_password": "newpass123",
    "confirm_password": "newpass123"
}

Response: {"message": "Password berhasil diubah"}

5. Profil Admin

GET /admin-profile Auth Required

Response: Lihat detail admin (email, nama, role, dll).

💬 Chatbot

Kirim Pertanyaan

POST /chat Auth Required (API Key)

Request Body: {"pertanyaan": "Apa menu favorit?"}

Response (sukses, jawaban ditemukan):

{
    "pertanyaan": "...",

Response (ambigu, beberapa kemungkinan):

👥 Admin Management (Super Admin Only)

Semua endpoint di bawah ini membutuhkan token admin dengan role super_admin. Headers yang diperlukan: Authorization: Bearer <token> dan X-API-Key.

1. Daftar Semua Admin

GET /api/admins?page=1&per_page=10&search=...

Response: JSON dengan pagination dan array data admin.

2. Tambah Admin Baru

POST /api/admins

Request Body:

{
    "username": "newadmin",
    "password": "pass123",
    "email": "new@royalsresto.com",
    "full_name": "Admin Baru",
    "role": "admin",       // atau "super_admin"
    "is_active": true
}

Response: {"message": "Admin berhasil dibuat", "admin_id": 5}

3. Update Admin

PUT /api/admins/{admin_id}

Request Body: (field yang ingin diubah)

{
    "email": "newemail@royalsresto.com",
    "full_name": "Nama Baru",
    "role": "admin",
    "is_active": true
}

4. Reset Password Admin

POST /api/admins/{admin_id}/reset-password

Request Body: {"new_password": "newpass123"}

5. Hapus Admin

DELETE /api/admins/{admin_id}

📊 Manajemen Dataset (Pertanyaan & Jawaban)

Dataset chatbot disimpan di database PostgreSQL (tabel dataset). Endpoint ini digunakan untuk mengelola pasangan pertanyaan-jawaban.

1. Ambil Semua Data (dengan pagination, filter)

GET /get-all-data?page=1&per_page=20&search=menu&kategori=harga Auth Required (API Key)

Query parameters: page, per_page, search, kategori (semua opsional).

Response:

{
    "page": 1,
    "per_page": 20,
    "total_data": 1250,
    "total_pages": 63,
    "data": [
        {
            "index": 0,
            "id": 1,
            "pertanyaan": "hai",
            "jawaban": "Hai, selamat datang...",
            "kategori": "introduction"
        }
    ]
}

2. Tambah Data Baru

POST /tambah-data

Request Body:

{
    "pertanyaan": "Apa itu AI?",
    "jawaban": "Artificial Intelligence...",
    "kategori": "teknologi"
}

Response Sukses: {"message": "Data berhasil ditambahkan", "status": "success"}

3. Update Data

PUT /update-data

Request Body: {"index": 0, "pertanyaan": "...", "jawaban": "...", "kategori": "..."}

4. Hapus Data (satu item berdasarkan index)

DELETE /delete-data

Request Body: {"index": 0}

5. Hapus Banyak Data

DELETE /delete-bulk-data

Request Body: {"indices": [0,2,5]}

6. Ambil Data Berdasarkan Index

GET /get-data/<index>

7. Export Dataset ke CSV

GET /api/export-data

Mendownload file CSV berisi seluruh dataset.

8. Pencarian Data

GET /api/search-data?q=kata&kategori=harga

Mengembalikan maksimal 50 hasil.

3. Hapus Semua Unknown

DELETE /delete-all-unknown

🤖 Training Model

⚠️ Karena lingkungan serverless (Vercel) memiliki batasan waktu dan memori, training model sebaiknya dilakukan secara lokal dan file model_qa.pkl di-commit ke repository. Endpoint berikut tetap tersedia untuk keperluan development atau jika dataset kecil.

1. Latih Model

POST /train-model Auth Required (API Key)

Membaca semua data dari database, melakukan preprocessing, TF-IDF vectorizing, dan melatih model SVM. Hasil model disimpan ke model/model_qa.pkl. Respons:

{
    "message": "Model berhasil dilatih",
    "training_time": "8.32 detik",
    "total_data": 1250,
    "categories_count": 25,
    "status": "success"
}

2. Informasi Model

GET /model-info Auth Required (API Key) 
{
    "total_questions": 1250,
    "categories": ["introduction","menu","harga",...],
    "model_loaded": true,
    "vectorizer_loaded": true
}

3. Daftar Kategori (dari dataset)

GET /kategori Auth Required (API Key) 
{"kategori": ["introduction","menu","harga","promo","fasilitas"]}
